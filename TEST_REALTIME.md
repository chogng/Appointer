# 实时同步功能测试指南

## 🎯 测试目标

验证 WebSocket 实时同步功能是否正常工作。

## 📋 准备工作

### 1. 启动后端服务器

```bash
cd server
npm install
node server.js
```

看到以下输出表示成功：
```
✅ 数据库已加载（或创建新数据库）
✅ 初始数据已插入
🚀 服务器运行在 http://localhost:3001
🔌 WebSocket 已启用
```

### 2. 启动前端开发服务器

**新开一个终端**，运行：

```bash
npm install
npm run dev
```

访问 `http://localhost:5173`

## 🧪 测试场景

### 场景 1：设备状态实时同步 ⚡

**目的**：验证管理员修改设备状态后，普通用户立即看到变化

**步骤**：

1. **准备两个浏览器窗口**
   - 窗口 A：Chrome 正常模式
   - 窗口 B：Chrome 隐私模式（或使用 Firefox）

2. **窗口 A - 管理员操作**
   ```
   登录账号：admin
   密码：123
   进入：设备列表页面
   ```

3. **窗口 B - 普通用户观察**
   ```
   登录账号：user
   密码：123
   进入：设备列表页面
   ```

4. **执行测试**
   - 在窗口 A（管理员）中，点击任意设备的开关
   - 观察窗口 B（用户）**无需刷新**，状态自动变化
   - 开关：启用 → 停用
   - 状态：🟢 可用 → 🔴 维护中

5. **验证成功标志**
   - ✅ 窗口 B 自动更新，无需刷新
   - ✅ 浏览器控制台显示：`📡 收到设备更新: {...}`
   - ✅ 状态变化流畅，无延迟

---

### 场景 2：预约实时同步 📅

**目的**：验证用户 A 预约后，用户 B 立即看到该时间段被占用

**步骤**：

1. **准备两个浏览器窗口**
   - 窗口 A：用户 A
   - 窗口 B：用户 B（或管理员）

2. **两个窗口都进入同一设备的预约页面**
   ```
   点击任意设备 → 点击"立即预约"按钮
   ```

3. **窗口 A - 创建预约**
   - 选择某个日期和时间段
   - 点击预约

4. **窗口 B - 观察变化**
   - **无需刷新**，该时间段自动变为"已预约"状态
   - 颜色变化或显示预约者信息

5. **验证成功标志**
   - ✅ 窗口 B 自动更新日历
   - ✅ 控制台显示：`📡 收到新预约（实时）: {...}`
   - ✅ 时间段状态立即变化

---

### 场景 3：多用户并发测试 👥

**目的**：验证多个用户同时操作时的同步效果

**步骤**：

1. **打开 3 个浏览器窗口**
   - 窗口 A：admin
   - 窗口 B：manager
   - 窗口 C：user

2. **所有窗口进入设备列表页面**

3. **在窗口 A 中快速切换多个设备状态**

4. **观察窗口 B 和 C**
   - 所有变化都实时同步
   - 无需任何手动刷新

5. **验证成功标志**
   - ✅ 所有窗口状态一致
   - ✅ 无延迟或丢失更新
   - ✅ 控制台显示多条广播消息

---

### 场景 4：断线重连测试 🔌

**目的**：验证网络断开后重连的稳定性

**步骤**：

1. **打开浏览器控制台（F12）→ Network 标签**

2. **勾选 "Offline" 模拟断网**

3. **观察控制台输出**
   ```
   ❌ WebSocket 已断开
   ```

4. **取消 "Offline"，恢复网络**

5. **观察控制台输出**
   ```
   ✅ WebSocket 已连接
   ```

6. **验证成功标志**
   - ✅ 自动重连成功
   - ✅ 重连后继续接收实时更新
   - ✅ 无需刷新页面

---

## 🐛 常见问题排查

### 问题 1：控制台没有 WebSocket 日志

**原因**：后端服务器未启动或端口错误

**解决**：
```bash
# 检查后端是否运行
curl http://localhost:3001/api/devices

# 如果失败，重启后端
cd server
node server.js
```

---

### 问题 2：状态不同步

**原因**：WebSocket 连接失败

**检查**：
1. 打开浏览器控制台 → Network → WS 标签
2. 查看是否有 WebSocket 连接
3. 检查连接状态是否为 "101 Switching Protocols"

**解决**：
- 确保后端运行在 3001 端口
- 检查 CORS 配置
- 清除浏览器缓存

---

### 问题 3：延迟很高

**原因**：网络问题或服务器负载
